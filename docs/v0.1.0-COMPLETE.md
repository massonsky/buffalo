### 1. pkg/logger - Система Логирования

**Файлов:** 9 | **Тестов:** 16 | **Покрытие:** 54.7%

#### Возможности:
- ✅ Структурированное логирование с Fields
- ✅ 5 уровней логирования (DEBUG, INFO, WARN, ERROR, FATAL)
- ✅ 3 форматтера: JSON, Text, Colored (ANSI)
- ✅ 2 типа вывода: Console (stdout/stderr), File
- ✅ Ротация логов (по размеру, возрасту, дням)
- ✅ Поддержка дочерних логгеров с наследованием Fields
- ✅ Options pattern для гибкой конфигурации

#### Файлы:
```
pkg/logger/
├── logger.go          # Основная структура Logger
├── level.go           # Уровни логирования
├── formatter.go       # Интерфейс Formatter
├── formatter_json.go  # JSON форматтер с PrettyPrint
├── formatter_text.go  # Текстовый форматтер
├── formatter_colored.go # Цветной форматтер (ANSI codes)
├── output.go          # Интерфейс Output
├── output_console.go  # Консольный вывод
└── output_file.go     # Файловый вывод с ротацией
```

#### Примеры использования:
```go
// Создание logger с цветным форматированием
log := logger.New(
    logger.WithLevel(logger.DEBUG),
    logger.WithFormatter(logger.NewColoredFormatter()),
)

// Structured logging
log.WithFields(logger.Fields{
    "component": "builder",
    "stage": "compilation",
}).Info("Starting compilation")

// Файловый вывод с ротацией
log := logger.New(
    logger.WithOutput(logger.NewFileOutput(
        "logs/buffalo.log",
        logger.WithMaxSize(10*1024*1024), // 10MB
        logger.WithMaxAge(7*24*time.Hour), // 7 дней
        logger.WithMaxBackups(5),
    )),
)
```

### 3. pkg/utils - Утилиты

**Файлов:** 5 | **Тестов:** 46 | **Покрытие:** 67.3%

#### Возможности:
- ✅ Файловые операции (поиск, копирование, чтение, запись)
- ✅ Работа с путями (нормализация, join, относительные пути)
- ✅ Хеширование (MD5, SHA1, SHA256, SHA512)
- ✅ Валидация proto файлов и имён пакетов
- ✅ Worker pool для параллельного выполнения
- ✅ Кроссплатформенная поддержка путей

#### Файлы:
```
pkg/utils/
├── file.go        # Файловые операции
├── path.go        # Работа с путями
├── hash.go        # Хеширование
├── validation.go  # Валидация proto файлов
└── worker.go      # Worker pool
```

#### Ключевые функции:

**file.go:**
- `FindFiles(dir, pattern string, recursive bool)` - поиск файлов
- `CopyFile(src, dst string)` - копирование
- `EnsureDir(path string)` - создание директории
- `CleanDir(path string)` - очистка директории
- `ReadFile/WriteFile` - чтение/запись

**path.go:**
- `NormalizePath(path string)` - нормализация
- `JoinPath(elements ...string)` - объединение путей
- `GetRelativePath(base, target string)` - относительный путь
- `ChangeExtension(path, ext string)` - смена расширения
- `ToSlash/FromSlash` - кроссплатформенные пути

**hash.go:**
- `ComputeHash(data []byte, algo string)` - хеширование данных
- `ComputeFileHash(path, algo string)` - хеш файла
- `HashFile(path string)` - SHA256 файла (по умолчанию)
- `VerifyFileHash(path, expected, algo string)` - проверка хеша

**validation.go:**
- `ValidateProtoFile(path string)` - валидация proto файла
- `ValidateProtoPackageName(name string)` - валидация имени пакета
- `ValidateOutputDir(path string)` - валидация выходной директории

**worker.go:**
- `NewWorkerPool(workerCount int)` - создание пула
- `Execute(tasks []Task)` - выполнение задач
- `ParallelExecute(tasks []Task, maxWorkers int)` - удобная обёртка

#### Примеры использования:
```go
// Поиск всех .proto файлов рекурсивно
files, _ := utils.FindFiles("./protos", "*.proto", true)

// Валидация proto файла
result := utils.ValidateProtoFile("user.proto")
if !result.IsValid() {
    for _, err := range result.Errors {
        log.Error(err)
    }
}

// Хеширование файла
hash, _ := utils.HashFile("example.proto") // SHA256

// Параллельное выполнение
pool := utils.NewWorkerPool(4)
results := pool.Execute([]utils.Task{
    {ID: "task1", Func: func() error { return compileProto("user.proto") }},
    {ID: "task2", Func: func() error { return compileProto("order.proto") }},
})
```

### 4. pkg/metrics - Метрики

**Файлов:** 3 | **Тестов:** 20 | **Покрытие:** 90.9%

#### Возможности:
- ✅ 3 типа метрик: Counter, Gauge, Histogram
- ✅ Thread-safe операции с atomic
- ✅ Collector для управления метриками
- ✅ Exporter с форматами: Text, JSON, Prometheus
- ✅ Глобальные labels для метрик
- ✅ Snapshots для захвата состояния

#### Файлы:
```
pkg/metrics/
├── collector.go  # Collector для управления метриками
├── metric.go     # Counter, Gauge, Histogram
└── exporter.go   # Экспорт в Text, JSON, Prometheus
```

#### Типы метрик:

**Counter** - Только увеличивается:
```go
counter := collector.Counter("builds_total")
counter.Inc()        // +1
counter.Add(5)       // +5
value := counter.Get()
```

**Gauge** - Может изменяться в обе стороны:
```go
gauge := collector.Gauge("active_workers")
gauge.Set(10)        // Установить значение
gauge.Inc()          // +1
gauge.Dec()          // -1
gauge.Add(5)         // +5
gauge.Sub(3)         // -3
value := gauge.Get()
```

**Histogram** - Распределение значений с buckets:
```go
hist := collector.Histogram("build_duration_ms", nil) // default buckets
hist.Observe(245.5)  // Добавить наблюдение
count := hist.Count()
sum := hist.Sum()
avg := hist.Average()
```

#### Collector:
```go
collector := metrics.NewCollector()

// Регистрация метрик
collector.Register("requests_total", metrics.NewCounter("requests_total"))

// Или через shortcuts
counter := collector.Counter("requests_total")
gauge := collector.Gauge("cpu_usage")
hist := collector.Histogram("latency_ms", []float64{10, 50, 100, 500})

// Глобальные labels
collector.SetLabel("service", "buffalo")
collector.SetLabel("version", "0.1.0")

// Snapshot
snapshot := collector.Snapshot()
fmt.Printf("Captured at: %v\n", snapshot.Timestamp)
```

#### Exporter:
```go
exporter := metrics.NewExporter(collector)
exporter.WithLogger(log) // Опционально

// Экспорт в разные форматы
exporter.Export(metrics.FormatText, os.Stdout)
exporter.Export(metrics.FormatJSON, jsonFile)
exporter.Export(metrics.FormatPrometheus, httpResponse)

// Логирование метрик
exporter.LogMetrics()
```

#### Prometheus формат:
```
# TYPE requests_total counter
requests_total{service="buffalo",version="0.1.0"} 1234

# TYPE build_duration_ms histogram
build_duration_ms_count{service="buffalo",version="0.1.0"} 42
build_duration_ms_sum{service="buffalo",version="0.1.0"} 10500
```

## 🐛 Известные Ограничения

### pkg/logger:
- Ротация файлов работает только при Write
- MaxAge проверяется при каждой записи (может быть неэффективно)
- Нет асинхронного логирования (будет в v0.3.0)

### pkg/utils:
- FindFiles может быть медленным для больших деревьев
- Worker pool не поддерживает приоритеты задач
- Валидация proto файлов базовая (только имя файла и размер)

### pkg/metrics:
- Histogram buckets фиксированные (нельзя изменить после создания)
- Нет автоматического сброса метрик
- Экспорт синхронный (может блокировать)

### Общие:
- Нет конфигурационных файлов (будет в v0.2.0)
- Нет CLI команд (будет в v0.2.0)
- Нет интеграции с protoc (будет в v0.3.0)
