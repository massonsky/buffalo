### 4. pkg/metrics - Система метрик (90.9% покрытие) ✅

**Файлы:** 3 файла
- `collector.go` - Коллектор метрик с регистрацией и управлением
- `metric.go` - Типы метрик (Counter, Gauge, Histogram)
- `exporter.go` - Экспорт в форматы Text, JSON, Prometheus

**Тесты:** 4 файла, 20 тестов
- `collector_test.go` - Тесты коллектора
- `metric_test.go` - Тесты типов метрик
- `exporter_test.go` - Тесты экспорта
- `example_test.go` - Примеры использования

**Counter (Счётчик):**
- `Inc()` - Увеличение на 1
- `Add(delta)` - Добавление значения (только положительные)
- `Get()` - Получение текущего значения
- `Reset()` - Сброс в 0

**Gauge (Индикатор):**
- `Set(value)` - Установка значения
- `Inc()`, `Dec()` - Увеличение/уменьшение на 1
- `Add(delta)`, `Sub(delta)` - Добавление/вычитание
- `Get()` - Получение значения
- `Reset()` - Сброс в 0

**Histogram (Гистограмма):**
- `Observe(value)` - Запись наблюдения
- `Count()` - Количество наблюдений
- `Sum()` - Сумма всех значений
- `Average()` - Среднее значение
- Настраиваемые buckets
- `Reset()` - Сброс

**Collector:**
- `Register(name, metric)` - Регистрация метрики
- `Get(name)` - Получение метрики
- `Counter(name)`, `Gauge(name)`, `Histogram(name, buckets)` - Удобные методы
- `All()` - Все метрики
- `Reset()` - Сброс всех метрик
- `SetLabel(key, value)` - Глобальные метки
- `Snapshot()` - Снимок всех метрик

**Exporter:**
- Форматы: Text, JSON, Prometheus
- `Export(format, writer)` - Экспорт метрик
- `LogMetrics()` - Логирование через logger
- Поддержка меток (labels)

## Интеграция компонентов

### main.go обновлён
Главный файл использует новый logger с цветным выводом:
```go
log := logger.New(
    logger.WithLevel(logger.INFO),
    logger.WithFormatter(&logger.ColoredFormatter{}),
    logger.WithOutput(&logger.ConsoleOutput{}),
)
```

### Взаимодействие компонентов
- **errors** использует коды ошибок, которые будут логироваться через **logger**
- **utils** использует **errors** для обработки ошибок файловых операций
- **metrics** может интегрироваться с **logger** для вывода метрик
- **worker** из utils обеспечивает параллелизм для будущих операций
